b.设置root用户的计划任务。每天早上7:50自动开启ssh服务,22:50关闭;每周六的7:30重新启动ssh服务
输入命令 crontab -e
50 7 * * * /usr/bin/service start sshd :每天早上7:50自动开启ssh服务
50 22 * * * /usr/bin/service stop sshd :22:50关闭

任务六 防火墙策略(Linux)

14.在工作时间,即周一周五的8:30-18:00,开放本机的ftp服务给192.168.1.0网络中的主机访问,
iptables -A INPUT -p tcp –dport 21 -s 192.168.1.0 -m time –weekdays 1,2,3,4,5
–timestart 8:30 –timestop 18:00 -j ACCEPT

15.要求从ftp 服务的数据下载请求次数每分钟不得超过5个;
可能是:
iptables -A INPUT -s 192.168.1.0 -p tcp -m state –state RELATED -m limit
–limit 5/min -j ACCEPT

16.拒绝TCP标志位全部位1及全部为0的报文访问本机;
iptables -A INPUT -p tcp –tcp-flags ALL ALL -j DROP
iptables -A INPUT -p tcp –tcp-flags ALL NONE -j DROP

17.配置iptables防火墙过滤规则,以封堵目录网段(172.16.1.0/24), 并在两个小时后解除封锁

输入crontab -e
iptables -D INPUT 1
iptables -D FORWARD 1

18.拒绝访问防火墙的新数据包,但允许响应连接或与已有连接相关的数据包
iptables -A INPUT -p tcp -m state –state NEW -j DROP
iptables -A INPUT -p tcp -m state –state ESTABLISHED,RELATED -j ACCEPT

任务一 登录安全加固(Windows Linux)
1.密码策略(Windows,Linux)
a.强制密码历史为5个密码;
Windows:
路径:

b.密码最长存留期为45天。
windows:

修改:PASSMAXDAYS 45

a.设置账号锁定阈值为3次错误锁定账号,锁定时间为30分钟,复位账号锁定计算器为30分钟之后

3.用户安全管理(Windows)
a.禁止从远端系统强制关机,将该权限只指派给administrator组;
在本地安全策略—>本地策略—>用户权限分配—>从远程系统强制关机

b.禁止发送未加密的密码到第三方SMB服务器
路径:在本地安全策略—>本地策略—>安全选项—>Microsoft 网络客户端:将未加密的密码发送到第三方SMB 选择禁用
